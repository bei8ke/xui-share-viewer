# x-ui 云端展示端 部署说明

> 本文档适合没有服务器运维经验的用户，请按步骤操作。

---

## 一、两种部署方案对比

| 方案 | 难度 | 费用 | 推荐程度 |
|------|------|------|---------|
| **方案 A：Manus 平台一键发布** | ⭐ 最简单 | 完全免费 | ⭐⭐⭐ 强烈推荐 |
| **方案 B：Railway 免费云平台** | ⭐⭐⭐ 较复杂 | 免费（有限额） | 适合需要独立服务器的用户 |

---

## 方案 A：Manus 平台一键发布（推荐）

这是最简单的方式，你不需要注册任何第三方账号，直接在 Manus 平台发布即可。

### 第一步：发布网站

在 Manus 对话界面右上角，找到 **Publish（发布）** 按钮，点击即可。

发布成功后，你会得到一个类似 `https://xxx.manus.space` 的网址，这就是你的云端展示网站地址。

### 第二步：登录管理后台

1. 在浏览器打开你的网站地址，例如 `https://xxx.manus.space`
2. 点击右上角「**管理员登录**」
3. 使用你的 Manus 账号登录（就是你平时用的账号）
4. 登录成功后，你会看到管理员后台界面

### 第三步：创建 API 密钥

API 密钥是本地工具向云端推送数据时用的「通行证」。

1. 在管理后台左侧菜单，点击「**API 密钥**」
2. 点击「**创建新密钥**」按钮
3. 填写一个名称（例如「本地工具」），点击确认
4. 系统会显示一串以 `xui_` 开头的密钥，**立即复制保存**，因为它只显示一次！

### 第四步：配置本地工具

将上一步复制的密钥填入本地工具的「同步到云端」对话框中：
- **云端服务器地址**：填写你的网站地址，例如 `https://xxx.manus.space`
- **API 密钥**：粘贴刚才复制的密钥

完成！现在可以从本地工具同步数据到云端了。

## 方案 B：Railway 免费云平台

如果你希望部署在独立的服务器上（不依赖 Manus 平台），可以使用 Railway。

> Railway 免费套餐每月提供 $5 的使用额度，对于小流量应用完全够用。

### 第一步：注册 Railway 账号

1. 打开 https://railway.app
2. 点击「Sign Up」，使用 GitHub 账号注册（需要先有 GitHub 账号）
3. 如果没有 GitHub 账号，先去 https://github.com 注册一个

### 第二步：注册 GitHub 账号并上传代码

1. 打开 https://github.com，注册账号
2. 登录后，点击右上角「+」→「New repository」
3. 仓库名称填写 `xui-share-viewer`，选择「Private（私有）」，点击「Create repository」
4. 按照页面提示，将本项目代码上传到这个仓库

   如果你不熟悉 Git，可以使用 GitHub Desktop：
   - 下载安装 https://desktop.github.com
   - 打开后登录 GitHub 账号
   - 点击「Add」→「Add Existing Repository」，选择本项目文件夹
   - 点击「Publish repository」上传

### 第三步：在 Railway 创建项目

1. 打开 https://railway.app，登录
2. 点击「New Project」
3. 选择「Deploy from GitHub repo」
4. 找到并选择你刚才创建的 `xui-share-viewer` 仓库
5. Railway 会自动检测并开始部署

### 第四步：添加数据库

1. 在 Railway 项目页面，点击「New」→「Database」→「MySQL」
2. 等待数据库创建完成（约 1 分钟）
3. 点击数据库，在「Variables」标签找到 `DATABASE_URL`，复制这个值

### 第五步：配置环境变量

在 Railway 项目的主服务（不是数据库）中，点击「Variables」标签，添加以下变量：

| 变量名 | 值 | 说明 |
|--------|-----|------|
| `DATABASE_URL` | 上一步复制的值 | 数据库连接地址 |
| `JWT_SECRET` | 随机字符串（32位以上） | 用于加密会话，可用密码生成器生成 |
| `OWNER_OPEN_ID` | 你的管理员标识 | 见下方说明 |

> **关于 OWNER_OPEN_ID：** 这是管理员账号的唯一标识。由于方案 B 使用 Manus OAuth，你需要先登录一次，然后在数据库中查看你的 `openId` 字段，将其填入此处。

### 第六步：初始化数据库表

Railway 部署完成后，需要手动创建数据库表。

1. 在 Railway 项目页面，点击数据库服务
2. 点击「Connect」→「MySQL Client」
3. 执行以下 SQL（复制粘贴运行）：

```sql
CREATE TABLE IF NOT EXISTS users (
  id INT AUTO_INCREMENT PRIMARY KEY,
  openId VARCHAR(64) NOT NULL UNIQUE,
  name TEXT,
  email VARCHAR(320),
  loginMethod VARCHAR(64),
  role ENUM('user','admin') NOT NULL DEFAULT 'user',
  createdAt TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updatedAt TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  lastSignedIn TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE IF NOT EXISTS records (
  id INT AUTO_INCREMENT PRIMARY KEY,
  panelId VARCHAR(255) NOT NULL,
  inboundId INT NOT NULL DEFAULT 0,
  remark VARCHAR(255),
  accelerateIp VARCHAR(255) NOT NULL,
  acceleratePort INT NOT NULL,
  vmessLink TEXT,
  clashLink TEXT,
  qrCodeUrl TEXT,
  protocol VARCHAR(50) DEFAULT 'vmess',
  batchId VARCHAR(64),
  status ENUM('success','failed','skipped') NOT NULL DEFAULT 'success',
  note TEXT,
  createdAt TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE IF NOT EXISTS groups_table (
  id INT AUTO_INCREMENT PRIMARY KEY,
  customerName VARCHAR(255) NOT NULL,
  uuidToken VARCHAR(64) NOT NULL UNIQUE,
  status ENUM('active','disabled') NOT NULL DEFAULT 'active',
  description TEXT,
  createdBy VARCHAR(255),
  createdAt TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updatedAt TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

CREATE TABLE IF NOT EXISTS group_records (
  id INT AUTO_INCREMENT PRIMARY KEY,
  groupId INT NOT NULL,
  recordId INT NOT NULL,
  createdAt TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY unique_group_record (groupId, recordId)
);

CREATE TABLE IF NOT EXISTS api_keys (
  id INT AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(255) NOT NULL,
  keyHash VARCHAR(64) NOT NULL UNIQUE,
  keyPrefix VARCHAR(20) NOT NULL,
  isActive BOOLEAN NOT NULL DEFAULT TRUE,
  lastUsedAt TIMESTAMP,
  usageCount INT NOT NULL DEFAULT 0,
  createdBy VARCHAR(255),
  createdAt TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```

### 第七步：获取网站地址

Railway 部署完成后，在项目页面点击「Settings」→「Domains」，可以看到自动分配的域名，格式类似 `xxx.railway.app`。

这就是你的云端展示网站地址，后续配置同方案 A 的第三、四步。

## 二、管理后台使用指南

部署完成后，以下是管理后台的日常使用方法。

### 2.1 数据总览

登录后台后，首页显示统计数据：
- 总记录数：所有已上传的操作结果数量
- 总分组数：已创建的客户分组数量
- 活跃分组：当前可访问的分组数量
- 未分配记录：还没有分配到任何分组的记录数量

### 2.2 查看所有记录

点击左侧「**所有记录**」，可以看到所有上传的操作结果。

支持按状态筛选：全部 / 已分配 / 未分配。

勾选多条记录后，可以点击「**创建分组**」按钮，将这些记录分配给一个客户。

### 2.3 管理分组

点击左侧「**分组管理**」，可以：
- 查看所有客户分组及其包含的记录数量
- 点击「查看详情」查看分组内的具体记录
- 点击「禁用」使该分组的链接失效（客户访问时显示「链接已失效」）
- 点击「启用」重新激活分组
- 点击「删除」永久删除分组（不影响记录本身）

### 2.4 管理链接

点击左侧「**链接管理**」，可以：
- 查看所有分组的专属访问链接
- 一键复制链接，发送给对应客户
- 在新标签页预览客户看到的页面
- 禁用/启用链接

### 2.5 管理 API 密钥

点击左侧「**API 密钥**」，可以：
- 创建新的 API 密钥（用于本地工具推送数据）
- 查看密钥的使用次数和最后使用时间
- 禁用不再使用的密钥
- 删除密钥

## 三、客户访问说明

客户收到链接后，直接在浏览器打开即可，无需登录，无需安装任何软件。

页面会显示该分组内所有节点的信息，包括：
- 节点名称和加速 IP/端口
- VMess 协议链接（可一键复制）
- Clash 订阅链接（可一键复制）
- 二维码（可扫码导入手机客户端）

如果链接不存在，页面显示「**链接无效**」。
如果链接已被禁用，页面显示「**链接已失效**」。

## 四、常见问题

**Q: 客户反映链接打不开？**
A: 检查该分组是否被禁用，在「链接管理」页面确认状态为「活跃」。

**Q: 本地工具同步失败，提示 401 Unauthorized？**
A: API 密钥无效或已被禁用。在「API 密钥」页面检查密钥状态，或重新创建一个新密钥。

**Q: 想更新某个客户的节点信息怎么操作？**
A: 在本地工具重新执行批量操作，同步时使用相同的客户分组名称，会自动创建新分组。旧分组可以在后台手动禁用。

**Q: Railway 部署后网站无法访问？**
A: 检查环境变量是否配置正确，特别是 `DATABASE_URL`。在 Railway 项目页面查看「Deployments」中的日志。
